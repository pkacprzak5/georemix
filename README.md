# 🌍 GeoRemix

**GeoRemix** is a research-driven guessing game created by the **BIT** Student Research Group in partnership with **NVIDIA**. It blends cutting-edge, AI-powered image processing with the tension of classic GeoGuessr-style exploration, inviting players to navigate unfamiliar worlds and pinpoint their locations.

Every panorama in GeoRemix was passed through a bespoke content pipeline that mixes generative models with custom ComfyUI nodes and in-house tooling. The workflow enhances street-level captures with depth-aware styling, edge detection, and theme-specific prompts, producing visually coherent worlds that still feel grounded in reality.

From the initial concept, through gameplay logic, UX, and deployment, the entire platform was designed and built by BIT Scientific Group members. GeoRemix embodies the team’s passion for creative technology, artificial intelligence, and accessible fun.

## 🎮 How to Play
1. **Launch the app.**
2. **Pick a nickname** so the backend can track your progress.
3. **Choose a round** — each contains five stylised locales generated by the AI pipeline.
4. **Explore the panorama** using mouse or touch to rotate the 360° view and hop between nodes.
5. **Open the minimap** to drop your guess.
6. **Lock it in** to see your score, the distance to the target, and a level recap.
7. **Climb the leaderboard** and challenge friends to beat your run.

## 🚀 Local Setup
1. **Clone the repository**

   ```bash
   git clone https://github.com/bit-labs/geo-remix.git
   ```

2. **Navigate to the project directory**

   ```bash
   cd geo-remix
   ```

3. **Create a `.env` file in the project root**

   ```env
   API_SECRET_KEY=change-me
   API_PORT=80
   API_HOST=http://localhost/api/
   DATABASE_URL=sqlite:///scores.db
   CLIENT_ORIGIN=http://localhost
   FLASK_ENV=production
   SKIP_API_KEY_CHECK=false
   ```

4. **Start the stack with Docker Compose**

   ```bash
   docker compose up --build
   ```

5. **Open the game** at `http://localhost` (the backend is proxied under `http://localhost/api`).

> 🔁 Rebuild with `docker compose up --build` after changing source files.

## 🧰 Project Structure
- `frontend/` — Vite + React + TypeScript client with shadcn-inspired UI, framer-motion transitions, and custom windowed layout components.
- `backend/` — Flask service serving panoramas, routes, leaderboard, and player management via SQLAlchemy.
- `pipeline/` — ComfyUI-integrated tooling that transforms raw panoramas into stylised scenes using prompt-driven workflows.
- `docker-compose.yaml` — Bridges nginx, frontend, and backend API into one deployment.

---

## 🖥️ Frontend (Vite + React + TypeScript)
- **Windowed UI shell** (`src/components/ui/window`) mimics retro desktop vibes while supporting drag + resize behaviour.
- **Navigation system** (`src/lib/navigation`) orchestrates cinematic transitions between modules like intro, gameplay, and results.
- **Gameplay components**
  - `PanoramaViewer` uses `@photo-sphere-viewer/virtual-tour-plugin` for server-driven node traversal.
  - `Minimap` couples Leaflet with MapLibre tiles for smooth map interaction.
  - `GameplayOverlay` manages timer, pause menu, and quick actions via an event bridge.
- **State orchestration**
  - `GameStateContext` wires together navigation, theme, data fetching, and event handling.
  - `DataSourceManager` caches leaderboards locally and wraps secure API calls with `VITE_API_SECRET_KEY`.
- **Styling** — Tailwind v4 + custom gradients, anime-like star overlays, and responsive typography tuned for large-format displays.

---

## 🐍 Backend (Flask + SQLAlchemy)
- **Core app** (`backend/app.py`)
  - Serves panoramas and metadata from `backend/images/`, `backend/thumbnails/`, and `backend/rounds/`.
  - Exposes REST endpoints for round metadata, node traversal, and score submission.
  - Configurable via environment variables (API key, origins, database URL, etc.).
- **Data layer** (`backend/models/database.py`)
  - `Player` and `RoundScore` models with automatic aggregation helpers (`overallScore`, `minDistance`, etc.).
  - Safe `init_database` guards against race conditions when multiple workers start.
- **Routes** (`backend/routes/scoring.py`)
  - Player CRUD (create, availability check, detail view).
  - Round score submission and leaderboard retrieval with time tiebreakers.
- **Security middleware** (`backend/middleware/security.py`)
  - API key enforcement (skippable in dev).
  - Origin validation and JSON payload checks.

---

## 🖼️ AI Remix Pipeline (`pipeline/`)
- **Entry point**: `process_localization.py`
  - Loads prompt JSON definitions (style, negative prompt, verification prompt).
  - Queues jobs against a ComfyUI server with different CFG / denoise / seed combinations.
  - Waits for queue drain, parses score files, and copies best batches to the final destination.
- **Workflow definition**: `workflows/pipeline_api.json`
  - Encodes the ComfyUI graph used to stylise panoramas, including edge detection, depth cues, and style infusion nodes.
- **Prompt packs**: `prompts/*.json`
  - Each theme (e.g., `van_gogh`, `moon`, `sci-fi`) has tailored positive/negative prompts and verification instructions.
- **Utilities** (`helpers.py`, `api_helpers.py`)
  - Queue management, output cleanup, score parsing, and image copying.
- **Logs** (`logs/*.log`)
  - Run-level logs for each city/style processed, useful for auditing generation runs.

For detailed pipeline setup, usage, and explanation, see `pipeline/README.md`.
